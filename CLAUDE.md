## Commands

```bash
# Run in development (hot reload — requires Wails CLI)
make dev
# or
wails dev

# Build Windows release binary
make build
# or
wails build -platform windows/amd64 -o dist/prime-migration-fyne.exe

# Install Wails CLI (once)
go install github.com/wailsapp/wails/v2/cmd/wails@latest

# Install frontend dependencies
cd frontend && npm install
```

**Wails CLI required** — Install via `go install`. On Windows, CGO/gcc is still needed for the Firebird driver (`nakagami/firebirdsql`). Use MSYS2 with `mingw-w64-ucrt-x86_64-gcc` and add `C:\msys64\ucrt64\bin` to PATH.

There are no automated tests in this codebase.

## Architecture

This is a desktop GUI application (Wails v2 + React/TypeScript) for migrating pharmacy data from legacy systems (FCERTA, PRISMA5) into Pharmacie. The module name is `github.com/primesoftwaresi/prime-migration-fyne`; the `go.mod` uses `replace` to alias `github.com/primesoftwaresi/prime-migration => .`, so the app and its `pkg/` library are the same module.

### Three-layer structure

**`main.go`** — Entry point at project root. Loads `.env`, resolves the Firebird history DSN, initialises SQLite (`db.InitConfigDB`), wires services, and starts Wails with an embedded React frontend.

**`internal/`** — Application layer:
- `internal/app/app.go` — Wails `App` struct. All exported methods become TypeScript bindings callable from React. Emits events (`migration:log`, `migration:status-update`, `migration:progress`, `migration:complete`, `history:status-changed`) via `runtime.EventsEmit` for real-time frontend updates.
- `internal/services/` — Four services: `AuthService` (HTTP POST to `nucleo.primesoftware.com.br`), `ConfigService` (CRUD wrapper over SQLite), `HistoryService` (Firebird central history DB facade), `MigrationService` (orchestrates migration options; delegates to `pkg/db`).
- `internal/models/` — Shared data structs (`UserSession`, `MigrationConfig`, `ClientConfig`, `DatabaseConfig`, migration mode constants).
- `internal/config/` — `LoadDotEnv()`, env-var readers for Firebird/Pharmacie form defaults (`app_env.go`), and `LoadHistoryDSN()`.

**`frontend/`** — React + TypeScript + Tailwind (Vite):
- `frontend/src/App.tsx` — State machine: login → main → migration views.
- `frontend/src/pages/LoginPage.tsx` — Login form; calls `Login()` Go binding.
- `frontend/src/pages/MainPage.tsx` — 3-tab form (Conexão / Opções / Migrar); calls config/migration Go bindings.
- `frontend/src/pages/MigrationPage.tsx` — Progress view; subscribes to migration events via `EventsOn`.
- `frontend/wailsjs/` — Auto-generated by `wails dev/build`. Contains Go method bindings (TypeScript) and the Wails runtime. **Do not edit manually** — stubs exist for local type-checking but are overwritten at build time.

**`pkg/`** — Core library (unchanged from previous version):
- `pkg/db/connect.go` — Opens a Firebird connection.
- `pkg/db/setup.go` — Deploys SQL procedures from `sps-fcerta/` or `sps-prisma5/`.
- `pkg/db/migrate.go` — `MigrarDados()` dispatcher.
- `pkg/db/prisma5/` — Pure-Go PRISMA5 migration logic.
- `pkg/db/historydb/` — Firebird schema DDL for central history DB.
- `pkg/db/configdb.go` — SQLite for local config persistence.
- `pkg/logger/` — Structured logger.

### Migration flow

1. User logs in → `App.Login()` → `AuthService` calls PrimeSoftware Nucleo API.
2. `MainPage` (3 tabs): Conexão (Firebird + Pharmacie credentials), Opções (entity checkboxes, Excel/vVencido extras), Migrar (summary + start).
3. `App.StartMigration(config)` saves config to SQLite, launches a goroutine, emits events as migration progresses.
4. `MigrationPage` subscribes to `migration:*` events for real-time log + status table updates.
5. **FCERTA** uses `EXECUTE PROCEDURE` SQL; **PRISMA5** uses pure Go in `pkg/db/prisma5/`. **Incremental mode is PRISMA5-only** and requires the central Firebird history DB.

### Configuration / secrets

Copy `.env.example` to `.env`. Key env vars:
- `PRIME_HISTORY_DSN` or `PRIME_HISTORY_*` parts — Firebird central history DB.
- `PRIME_FIREBIRD_*` — Pre-fills Firebird origin form fields.
- `PRIME_PHARMACIE_*` — Pre-fills Pharmacie destination form fields.
- `PRIME_CONFIG_DB_PATH` — Override SQLite path.
- `GO_PROJECT_DIR` — Needed when CWD differs from repo root.

`central.json` (not versioned) is an alternative to `.env` for the history DSN: `{"history_dsn": "..."}`.
